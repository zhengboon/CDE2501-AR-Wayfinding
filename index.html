Project Overview

This repository contains a Unity-based AR wayfinding MVP focused on a Queenstown estate test area. The app prioritizes safer, cognitively simpler guidance for elderly users, with route weighting for slope, stairs, clutter, darkness, narrow width, and rain exposure.

Troubleshooting

No movement / no route updates

Confirm Assets/Scenes/Main.unity is open, Play mode is active, and Sim Mode is enabled in the Quick Start panel.

Street View stays blank

Current manifest has no usable nodes. Generate new imagery and verify street_view_manifest.json node count is greater than zero.

GPS/Compass not ready on laptop

Use status lines in Quick Start overlay. They now show detailed sensor diagnostics (OS-disabled, initializing, accuracy unavailable, etc).

Pitch key triggers recalc

Manual recalc in simulation now requires Ctrl+R. You can still use F5 for recalc without conflicting with movement controls.

Cached launcher cannot find Unity

Set unityExecutable inside scripts/unity_cached_builder_config.json or use UNITY_EXE environment variable.

NUS map not using Google style

Set GOOGLE_MAPS_API_KEY and enable billing + Map Tiles API for that key; otherwise Google tile session returns 403 and the script falls back to OSM tiles.
