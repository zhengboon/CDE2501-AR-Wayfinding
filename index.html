Quick Start

Get up and running locally with the Wayfinding ecosystem in minutes.

1

Environment Setup

Install Unity Hub and Editor 2022.3.62f3 with Android/iOS modules.

2

Cached Build Launch

Run the automated script to utilize our fingerprint cache system for rapid iterative testing.

python scripts/unity_cached_builder.py --launch-editor
3

Generate Maps from KML

Build the map atlases automatically using our data generation pipeline.

build_engineering_nus_map.bat
4

Alpha Testing — Record Data

Build the APK, install on a tester's phone, and use the Rec: ON/OFF toggle in the Quick Start overlay to gather GPS & route telemetry as CSV.

python scripts/unity_cached_builder.py --force --target Android --output Builds/Android/CDE2501-Wayfinding.apk

Troubleshooting

Common issues and straightforward solutions.

Route does not update while moving

Check: Sim Mode is true and Main Camera is present.
Action: Press F5 for manual recalc, then move with WASD.

Play mode shows only sky/ground

Check: Active scene might be Untitled.
Action: Stop Play mode, open Assets/Scenes/Main.unity, and hit Play again.

GPS & Compass show "not ready"

Check: Device sensor state overlays.
Action: On laptop, toggle simulation mode F2. On mobile, enable Location Services and calibrate the device compass.

Minimap elements break when turning (follow heading)

Check: Follow-heading mode is active and the user rotates.
Action: Interaction mapping fix is applied (click/hover/zoom/pan now align with heading). If any visual overlay drift remains, toggle follow-heading off as a temporary fallback.
